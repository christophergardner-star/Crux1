# Contributing to Cruxy Stability Engine

Thank you for your interest in contributing to the Cruxy Stability Engine! We welcome contributions from the community to help make adaptive optimization accessible to everyone.

## Code of Conduct

This project adheres to a Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub. Include:
- A clear description of the bug.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Your environment (OS, Python version, PyTorch version).

### Suggesting Enhancements

We love new ideas! If you have a feature request, open an issue and tag it as an "enhancement". Describe the feature and why it would be useful.

### Pull Requests

1.  **Fork the repository** and create your branch from `main`.
2.  **Install dependencies**:
    ```bash
    pip install -e .
    pip install pytest
    ```
3.  **Make your changes**. Ensure your code is clean and commented.
4.  **Run tests**:
    ```bash
    pytest tests/
    ```
5.  **Submit a Pull Request**. Provide a clear description of your changes and reference any related issues.

## Development Guidelines

- **Style**: We follow PEP 8 guidelines.
- **Testing**: All new features must include unit tests.
- **Documentation**: Update the README or docstrings if you change functionality.
